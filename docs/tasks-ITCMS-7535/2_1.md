# Task 2.1 — Add `sort=brickInscription` to Default Fetch

**Phase:** 2 — Sorting + Pagination
**Spec:** [§3.1 Alphabetical Sorting](../03-features.md#31-alphabetical-sorting-az-default)

---

## Implementation Details

Add the `sort=brickInscription` query parameter to the default Drupal browse `fetchBricks()` call in `TheBricks.vue` so bricks display in A–Z order by inscription.

**Current state:** `fetchBricks()` builds a Drupal URL with `page[limit]` and `page[offset]`, and may still include transitional keyword-filter logic from the pre-Searchstax implementation. There is no explicit `sort` parameter, so browse results arrive in whatever default order Drupal returns.

**Changes:**

1. Append `&sort=brickInscription` to the fetch URL in `fetchBricks()`.
2. The sort parameter should be present for default browse requests, and should also be used for any Drupal-backed park-location filtering added before Searchstax ships.
3. Keyword search is not a hard Phase 2 requirement. If transitional Drupal keyword filtering still exists, treat it as temporary and do not let it drive the Phase 2 acceptance criteria.

**Implementation:**

```typescript
// In TheBricks.vue → fetchBricks()
const url = this.apiUrl +
  `bricks?page[limit]=20` +
  `&sort=brickInscription` +
  `&page[offset]=${offset}`;
```

**Files to modify:**

- `src/components/TheBricks.vue` — add sort parameter to fetch URL
