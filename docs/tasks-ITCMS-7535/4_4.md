## Implementation Details

Add an active-filter pills bar and "Clear all" button to the bottom of `BrickFilter.vue`. This provides visual feedback showing which filters are active and allows quick removal.

**Current state (after Tasks 4.2–4.3):** BrickFilter has keyword input and location list. No visual indicator of active filters beyond the input value and the highlighted list item.

**Target state (from XD wireframe):**

```
┌──────────────────────────────────────────────────┐
│ [Brick Inscription: sample keyword ×]  [Clear all] │
│ [Brick Location: Class Walk of 2019 ×]             │
└──────────────────────────────────────────────────┘
```

The pills bar sits below the keyword input and location list, inside the same `<form>` container. Per the wireframe PNG, the pills/buttons sit inside their own translucent white action strip rather than directly on the green panel.

This action row is also the only reset affordance for keyword filters. There is no separate inline clear button inside the text field.
The translucent strip itself stays visible even when inactive so the panel does not jump vertically when the buttons appear.

> **Follow-up note:** The current Phase 4 implementation keeps the strip visible and hides `Clear all` when no filters are active. The latest UX direction is to keep `Clear all` visible in that idle state as a disabled button, then enable it once a keyword or location is active. Record that as pending UX-finalization work rather than changing the implementation spec in this task yet.

**Pill rendering logic:**

```typescript
// Computed
get activeFilters(): Array<{ type: string; label: string; value: string }> {
  const filters = [];
  if (this.inscription) {
    filters.push({
      type: 'inscription',
      label: 'Brick Inscription',
      value: this.inscription,
    });
  }
  if (this.locationIds.length === 1) {
    const loc = this.locations.find(l => l.id === this.locationIds[0]);
    filters.push({
      type: 'location',
      label: 'Brick Location',
      value: loc?.name ?? this.locationIds[0],
    });
  } else if (this.locationIds.length > 1) {
    filters.push({
      type: 'location',
      label: 'Brick Locations',
      value: `${this.locationIds.length} selected`,
    });
  }
  return filters;
}
```

**Pill markup:**

```html
<!-- Section C: Active Filters — only renders when filters are active -->
<div v-if="activeFilters.length > 0" class="tw-mt-4 tw-flex tw-flex-wrap tw-items-center tw-gap-2">
  <!-- translucent white action strip -->
  <span
    v-for="filter in activeFilters"
    :key="filter.type"
    class="
      tw-inline-flex tw-items-center tw-gap-1
      tw-bg-white tw-text-black
      tw-font-oswald tw-text-sm
      tw-px-3 tw-py-1 tw-rounded-[23px]
    "
  >
    {{ filter.label }}: {{ filter.value }}
    <button
      type="button"
      class="
        tw-inline-flex tw-h-[20px] tw-w-[20px] tw-items-center tw-justify-center
        tw-rounded-full tw-bg-black tw-text-[20px] tw-leading-none tw-text-white
        focus:tw-outline-none
      "
      :aria-label="`Remove ${filter.label} filter`"
      @click="removeFilter(filter.type)"
    >
      ×
    </button>
  </span>

  <button
    type="button"
    class="
      tw-font-oswald tw-text-black tw-text-sm tw-font-semibold
      tw-bg-white tw-px-3 tw-py-1 tw-rounded-[23px]
      hover:tw-bg-gray-100
      tw-transition-colors tw-duration-200
    "
    @click="$emit('clearAll')"
  >
    Clear all
  </button>
</div>
```

**Methods:**

```typescript
removeFilter(type: string) {
  if (type === 'inscription') {
    this.$emit('update:inscription', '');
  } else if (type === 'location') {
    this.$emit('update:locationIds', []);
  }
},
```

**Styling notes:**

- Pill background: `tw-bg-brickLightGreen` (olive/green tone matching the wireframe).
- Action strip background: translucent white (`rgba(255,255,255,0.53)`) behind the chips/buttons.
- Pill background: solid white with `23px` radius.
- Pill text: black, Oswald.
- `×` close button: white `x` centered on a black `20px` circle.
- "Clear all" button: black text on a white Oswald button with `23px` radius, matching the wireframe.
- Pills wrap to multiple lines if both filters are active on narrow viewports.

## Findings

- The pills and clear-all controls were implemented in the same `BrickFilter.vue` rewrite as the location list so the filter component contract only changed once.
- The wireframe PNG was used as the source of truth for the action row, so the chips and `Clear all` now sit inside a semi-transparent white strip with black-on-white buttons rather than the older green-pill treatment.
- The wireframe PNG was also used for the chip/button details: Oswald text, `23px` rounded corners, and a black close-circle with a centered white `x`.
- The form-level action row also replaces the old inline keyword clear button, so reset behavior lives in one place.
- The translucent action strip now stays visible at all times so the filter panel keeps a stable visual structure even with no active filters.
- A follow-up UX note is still open: once finalized, `Clear all` should remain visible in the idle state as a disabled control rather than being hidden when no filters are active.
- The `Showing X bricks` counter was removed because the latest design does not use it and its appearance caused visual movement in the panel.
- Verified after implementation with `npm run lint` and `npm run typecheck`.
