## Implementation Details

Create `src/components/AppHeader.vue` — a stateless, presentational component with no props or emits.

**Structure:**

```
┌─────────────────────────────────────────────────────────────────┐
│  BABSON COLLEGE                                                 │
│  FIND MY BRICK AT KERRY MURPHY HEALEY PARK                     │
└─────────────────────────────────────────────────────────────────┘
```

**Markup outline:**

```html
<header class="tw-bg-brickBabsonGreen tw-w-full tw-py-4 tw-px-6">
  <div class="tw-max-w-brickMWL tw-mx-auto">
    <h1 class="tw-font-oswald tw-text-white tw-uppercase tw-text-2xl tw-font-bold">
      Babson College
    </h1>
    <p class="tw-font-oswald tw-text-white tw-uppercase tw-text-sm">
      Find My Brick at Kerry Murphy Healey Park
    </p>
  </div>
```

**Key points:**

- Full-width dark green bar (`tw-bg-brickBabsonGreen`).
- Line 1: "BABSON COLLEGE" — large, bold, uppercase, white, Oswald.
- Line 2: "FIND MY BRICK AT KERRY MURPHY HEALEY PARK" — smaller, uppercase, white, Oswald.
- No "View Locations" button here — that lives in AppHero (see Task 1.2).
- Inner content constrained by `tw-max-w-brickMWL tw-mx-auto` for consistent alignment with the rest of the page.

## Findings

- Added `tw-m-0` to both `<h1>` and `<p>` to reset default browser margins that could add unwanted spacing inside the header bar. Without this, some browsers add top/bottom margin to headings and paragraphs, which would throw off the vertical padding.
- The component is not yet wired into `App.vue` — that happens in Task 1.4 (restructure `App.vue` layout). The remaining acceptance criteria (responsive rendering, visual match) require the component to be placed in the full page layout and verified visually or via E2E tests.
- Unit tests cover: element rendering, text content, Tailwind class application (background, font, text color, width constraint), and stateless contract (no props, no emits). 7 tests total in `AppHeader.spec.ts`.
