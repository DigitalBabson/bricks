# Task 3.3 — Add 500ms Debounce to Keyword Search in TheBricks

**Phase:** 3 — SearchStax Integration
**Spec:** [§5 Phase 3 step 5](../05-phases.md)

---

## Implementation Details

Add a 500ms debounce to the keyword `inscription` watcher in `TheBricks.vue`. This prevents firing a SearchStax request on every keystroke and is required before wiring the SearchStax path.

**Current state:** The `inscription` watcher in TheBricks fires `fetchBricks()` immediately on every change when `value.length >= 3`. There is no debounce.

**Approach:** Use a simple timer-based debounce inside the watcher. No external library needed.

**Changes to `TheBricks.vue`:**

1. Add a `searchTimeout` property to `data()`:

```typescript
data() {
  return {
    bricks: [] as Brick[],
    currentPage: 1,
    totalPages: 1,
    pageSize: 20,
    showMessage: false,
    searchTimeout: null as ReturnType<typeof setTimeout> | null,
  }
}
```

2. Update the `inscription` watcher:

```typescript
inscription(value: string) {
  this.currentPage = 1;

  // Clear any pending debounced search
  if (this.searchTimeout) {
    clearTimeout(this.searchTimeout);
    this.searchTimeout = null;
  }

  if (value.length === 0) {
    // Clearing the search — fetch default list immediately (no debounce)
    this.fetchBricks();
  } else if (value.length >= 3) {
    // Debounce keyword search (SearchStax or Drupal CONTAINS)
    this.searchTimeout = setTimeout(() => {
      this.searchTimeout = null;
      this.fetchBricks();
    }, 500);
  }
  // 1-2 chars: do nothing (wait for more input)
},
```

3. Clean up timer on unmount — add `beforeUnmount` lifecycle hook:

```typescript
beforeUnmount() {
  if (this.searchTimeout) {
    clearTimeout(this.searchTimeout);
  }
},
```

**Design notes:**

- Clearing the input (`value.length === 0`) fires immediately with no debounce. The user expects instant feedback when they clear the search box.
- The `locationIds` watcher remains immediate (no debounce) since location selection is a discrete click, not continuous typing.
- The debounce timer is per-watcher-invocation: each new keystroke resets the 500ms window.

**Files to modify:**

- `src/components/TheBricks.vue` — add `searchTimeout` data property, update `inscription` watcher, add `beforeUnmount` cleanup

## Acceptance Criteria

- [ ] Typing in the keyword field does not trigger a fetch until 500ms after the last keystroke.
- [ ] Clearing the keyword field triggers an immediate fetch (no debounce).
- [ ] Keywords under 3 characters do not trigger any fetch.
- [ ] Rapid typing (e.g., "bru" then quickly "bruce") only fires one fetch for "bruce".
- [ ] The `locationIds` watcher is not affected (still fires immediately).
- [ ] Timer is cleaned up on component unmount.
- [ ] Page resets to 1 immediately on watcher trigger (before debounce wait).
- [ ] `npm run lint` and `npm run typecheck` pass.
- [ ] Existing pagination and location-only filtering are not affected.
