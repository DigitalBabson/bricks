## Implementation Details

Add TypeScript types for the SearchStax response, create new injection keys for the SearchStax endpoint and token, and provide them from `main.ts`. This is the foundation all other Phase 3 tasks depend on.

**Current state:** `main.ts` provides `defaultEnvKey` (Drupal origin) and `defaultUrlKey` (Drupal JSON:API endpoint). The `.env` already contains `DEV_SEARCHSTAX_ENDPOINT` and `DEV_SEARCHSTAX_TOKEN`, and `env.d.ts` already declares their types. No SearchStax-specific injection keys or response types exist yet.

**New types â€” add to `src/types/index.ts`:**

```typescript
// SearchStax response types

export interface SearchstaxDoc {
  ss_uuid: string                         // Brick media entity UUID (matches Brick.id from Drupal)
  ss_zone_uuid: string                    // Park location UUID (matches Brick.brickParkLocation)
  ss_file_img_uuid: string                // File entity UUID for brick image (matches Brick.brickImage)
  ss_body: string                         // Park location name (informational, not used for filtering)
  'tcngramm_X3b_en_description': string[] // Inscription text (array of strings)
  'tm_X3b_en_title': string[]             // Title (same as inscription)
  its_fid: number                         // Drupal file ID (not used by frontend)
  its_id: number                          // Drupal media entity numeric ID
}

export interface SearchstaxResponse {
  response: {
    numFound: number
    start: number
    docs: SearchstaxDoc[]
  }
}
```

**New injection keys â€” add to `src/types/index.ts`:**

```typescript
export const searchstaxEndpointKey: InjectionKey<string> = Symbol('searchstaxEndpoint')
export const searchstaxTokenKey: InjectionKey<string> = Symbol('searchstaxToken')
```

**Update `src/main.ts`:**

Read SearchStax env vars and provide them alongside the existing Drupal keys:

```typescript
const searchstaxEndpoint = (import.meta.env.DEV_SEARCHSTAX_ENDPOINT ?? '').trim()
const searchstaxToken = (import.meta.env.DEV_SEARCHSTAX_TOKEN ?? '').trim()

app.provide(searchstaxEndpointKey, searchstaxEndpoint)
app.provide(searchstaxTokenKey, searchstaxToken)
```

**Note:** The existing Drupal endpoint cutover from hardcoded `intranet.babson.edu` to `.env` was already completed during Phase 1 (Task 1.7). The `main.ts` fallback line remains as a safety net for builds without a `.env` file but is not exercised in practice.
