# Task 6.1 — Add `isComingSoon` + Inscription Overlay to `BrickCard.vue`

**Phase:** 6 — BrickCard Enhancements
**Spec:** [§3.4 "Coming Soon" Bricks — Show Inscription](../03-features.md#34-coming-soon-bricks--show-inscription)
**Depends on:** Existing BrickCard placeholder-image behavior from Phase 1

---

## Implementation Details

Add a dedicated "Coming Soon" rendering path for bricks that do not yet have a photo. The card should continue using the shared placeholder image, but the image area must also display the brick inscription and the "Image Coming Soon" copy as an overlay.

**Current state:** `BrickCard.vue` falls back to `DEV_PLACEHOLDER_IMAGE` / `coming-soon-gray.jpg` when `brick.brickImage === 'default'`, but it does not visually identify which inscription belongs to that placeholder. Users can only identify the brick by opening the location-details modal.

**Target behavior:**

- `isComingSoon` is computed from the brick data, not from a failed image request.
- The placeholder image remains the background.
- The inscription text is overlaid in the image area using uppercase Oswald styling.
- "Image Coming Soon" appears beneath the inscription.
- The overlay is rendered only for placeholder bricks.

### `BrickCard.vue` changes

Add a computed property:

```typescript
isComingSoon(): boolean {
  return !this.brick.brickImage || this.brick.brickImage === 'default'
}
```

Render the image area as a positioned container:

```html
<div class="brick-card__media tw-relative">
  <img ... />

  <div
    v-if="isComingSoon"
    class="
      tw-absolute tw-inset-0
      tw-flex tw-flex-col tw-items-center tw-justify-center
      tw-text-center tw-px-4
    "
  >
    <p class="tw-font-oswald tw-uppercase tw-text-brickSummerNight tw-text-lg tw-font-semibold">
      {{ brick.inscription }}
    </p>
    <p class="tw-mt-2 tw-font-zilla tw-text-brickBabsonGrey">
      Image Coming Soon
    </p>
  </div>
</div>
```

**Implementation notes:**

- Reuse the existing placeholder-image fetch logic; do not introduce a second placeholder asset path.
- `isComingSoon` should drive the overlay immediately, before `img` load/error events fire.
- The inscription copy may be long, so the overlay needs centered text and horizontal padding.
- Keep the placeholder image visible beneath the overlay so the existing visual contract remains intact.

**Files to modify:**

- `src/components/BrickCard.vue`
