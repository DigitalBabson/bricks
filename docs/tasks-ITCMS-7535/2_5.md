## Implementation Details

Write unit tests for the rewritten `Pagination.vue` and update existing `TheBricks` tests to cover the new sort/pagination behavior.

### Pagination.vue Tests

Create `src/components/__tests__/Pagination.spec.ts`:

**Rendering tests:**

- Renders `<nav>` with `aria-label="Page navigation"`.
- Renders the correct number of page buttons for a small page count (e.g., `totalPages=3`).
- Does not render when `totalPages <= 1`.
- Active page has `aria-current="page"` and the active class (`tw-bg-brickBabsonGreen`).
- Inactive pages have `tw-text-brickSummerNight`.
- Previous arrow is disabled on page 1.
- Next arrow is disabled on the last page.

**Ellipsis tests:**

- No ellipsis when `totalPages <= maxVisible + 2`.
- Shows trailing ellipsis + last page when `currentPage` is near the start (e.g., page 2 of 50).
- Shows leading ellipsis + first page when `currentPage` is near the end (e.g., page 49 of 50).
- Shows both leading and trailing ellipsis when `currentPage` is in the middle (e.g., page 25 of 50).

**Emit tests:**

- Clicking a page number emits `update:page` with that number.
- Clicking `<` emits `update:page` with `currentPage - 1`.
- Clicking `>` emits `update:page` with `currentPage + 1`.
- Clicking a disabled arrow does not emit.

### TheBricks.vue Test Updates

Update `src/components/__tests__/TheBricks.spec.ts` (or create if it doesn't exist):

**Sorting:**

- `fetchBricks()` URL includes `&sort=brickInscription`.

**Pagination state:**

- Initial state has `currentPage: 1`, `totalPages: 1`.
- After fetch, `totalPages` is calculated from `meta.count`.
- `goToPage(3)` sets `currentPage` to 3 and triggers a fetch with `page[offset]=40`.
- Bricks array is replaced (not appended) when page changes.

**Drupal query reset:**

- Changing the active Drupal-backed query resets `currentPage` to 1.

**Template:**

- Pagination component receives `:currentPage` and `:totalPages` props.
- Pagination is hidden when `totalPages <= 1`.
