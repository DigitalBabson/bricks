## Implementation Details

Add `locationId` state to the data flow and wire it into `TheBricks.vue` so that selecting a park location filters the brick results.

**Current state:** `TheBricks.vue` receives `inscription` as a prop from `App.vue` and fetches bricks via a single Drupal URL. There is no location filter parameter.

**Changes to `App.vue`:**

1. Add `locationId: ''` to `App.vue` data (alongside existing `inscription`).
2. Pass `locationId` down to `TheBricks` as a prop.
3. Bind `v-model:locationId` on `BrickFilter` (in the hero slot).
4. Pass `locations` to `BrickFilter` as a prop.
5. Handle the `clearAll` emit from BrickFilter (reset both `inscription` and `locationId` to empty).

```html
<!-- App.vue template -->
<app-hero @openLocations="showLocationExplorer = true">
  <brick-filter
    v-model:inscription="inscription"
    v-model:locationId="locationId"
    :locations="locations"
    :resultCount="totalCount"
    @clearAll="clearAllFilters"
  />
</app-hero>
<the-bricks
  :inscription="inscription"
  :locationId="locationId"
  @update:totalCount="totalCount = $event"
/>
```

```typescript
// App.vue data (add totalCount)
data() {
  return {
    showLocationExplorer: false,
    inscription: '',
    locationId: '',
    locations: [] as ParkLocation[],
    totalCount: 0,
  }
},

// App.vue methods
clearAllFilters() {
  this.inscription = '';
  this.locationId = '';
},
```

**Changes to `TheBricks.vue`:**

1. Add `locationId` prop: `{ type: String, default: '' }`.

2. Add a watcher on `locationId` that resets `currentPage` to 1 and re-fetches:

   ```typescript
   watch: {
     inscription(value: string) {
       this.currentPage = 1;
       if (value.length === 0 || value.length >= 3) {
         this.fetchBricks();
       }
     },
     locationId() {
       this.currentPage = 1;
       this.fetchBricks();
     },
   },
   ```

3. Update `fetchBricks()` to add the location filter branch. Phase 4 only handles the two Drupal-backed paths (location-only and default browse). Keyword search paths (keyword-only and combined keyword + location) are owned entirely by Phase 3.

   ```typescript
   async fetchBricks() {
     try {
       const offset = (this.currentPage - 1) * this.pageSize;
       const locId = this.locationId;

       let url: string;

       if (locId) {
         // LOCATION ONLY → Drupal with park location filter
         url = this.apiUrl +
           `bricks?page[limit]=${this.pageSize}` +
           `&sort=brickInscription` +
           `&filter[brickParkLocation.id]=${locId}` +
           `&page[offset]=${offset}`;
       } else {
         // DEFAULT BROWSE → Drupal (with optional CONTAINS keyword filter)
         url = this.apiUrl +
           `bricks?page[limit]=${this.pageSize}` +
           `&sort=brickInscription` +
           `&page[offset]=${offset}`;
       }

       const response = await axios.get<BrickApiResponse>(url);
       // ... existing response handling ...
     } catch { /* ... */ }
   }
   ```

4. Update the empty-results message logic: show "No bricks match your criteria" when **any** active filter returns zero results (not just keyword).

5. `fetchBricks()` no longer takes a `search` parameter — it reads `this.inscription` and `this.locationId` directly from props/state. This simplifies the method signature and makes the routing logic clearer.

**`totalCount` contract:**

TheBricks emits `update:totalCount` after every fetch so App.vue can pass the value down to BrickFilter's `resultCount` prop. This follows the same prop-down/emit-up pattern used by `inscription` and `locationId`.

**TheBricks.vue:**

```typescript
emits: ['update:totalCount'],

// Inside fetchBricks(), after parsing the response:
const totalCount = response.data.meta?.count ?? 0;
this.totalPages = Math.ceil(totalCount / this.pageSize) || 1;
this.$emit('update:totalCount', totalCount);
```

**App.vue — data:**

```typescript
data() {
  return {
    showLocationExplorer: false,
    inscription: '',
    locationId: '',
    locations: [] as ParkLocation[],
    totalCount: 0,
  }
},
```

**App.vue — template:**

```html
<the-bricks
  :inscription="inscription"
  :locationId="locationId"
  @update:totalCount="totalCount = $event"
/>
```

The `:resultCount="totalCount"` binding on BrickFilter (shown in the template above) completes the chain.
