## Implementation Details

Replace the current "Load more" button with a full numbered pagination control matching the XD wireframe: `< 1 2 3 4 5 ... 50 >`.

**Current state:** `Pagination.vue` renders a single "Load more" button that emits `loadmore`. No page numbers, no awareness of total pages.

**New component contract:**

**Props:**

| Prop | Type | Default | Description |
|---|---|---|---|
| `currentPage` | `Number` | — (required) | Active page (1-indexed) |
| `totalPages` | `Number` | — (required) | Total number of pages |
| `maxVisible` | `Number` | `5` | Max numbered buttons before ellipsis |

**Emits:** `@update:page` with the target page number (1-indexed).

**Rendered output:**

```
< 1  2  3  4  5  ...  50  >
      ─
```

**Behavior:**

- `<` (previous) and `>` (next) arrow buttons at the ends.
- Use **dynamic ellipses**:
  - near the start: first `maxVisible` page numbers, then `...`, then the last page number
  - in the middle: first page, `...`, current page window, `...`, last page
  - near the end: first page, `...`, then the last `maxVisible` page numbers
- Current page is visually highlighted (underline or background).
- Arrows disabled at boundaries: page 1 disables `<`, last page disables `>`.
- Clicking a page number emits `@update:page` with that number.
- Clicking `<` emits `@update:page` with `currentPage - 1`.
- Clicking `>` emits `@update:page` with `currentPage + 1`.
- Component does not render if `totalPages <= 1`.

**Styling (Tailwind with `tw-` prefix):**

- Font: `tw-font-oswald`.
- Active page: `tw-bg-brickBabsonGreen tw-text-white` (or underlined variant).
- Inactive pages: `tw-text-brickSummerNight` with `hover:tw-underline`.
- Arrows: `tw-text-brickBabsonGreen`; disabled state: `tw-opacity-30 tw-cursor-not-allowed`.
- Overall container: centered, with `tw-mt-brick9 md:tw-mt-brick35 lg:tw-mt-brick50` spacing (matching current Pagination margins).

**Accessibility:**

- Outer element: `<nav aria-label="Page navigation">`.
- Active page button: `aria-current="page"`.
- Arrow buttons: `aria-label="Previous page"` / `aria-label="Next page"`.
- Disabled arrows: `disabled` attribute (not just visual).

**Ellipsis logic (computed):**

```typescript
// Pseudocode for visible pages
if (totalPages <= maxVisible + 2) {
  // Show all pages — no ellipsis needed
  pages = [1, 2, ..., totalPages]
} else if (currentPage <= maxVisible) {
  // Near the start: show first maxVisible, ..., last
  pages = [1, 2, ..., maxVisible, '...', totalPages]
} else if (currentPage > totalPages - maxVisible) {
  // Near the end: show first, ..., last maxVisible
  pages = [1, '...', totalPages - maxVisible + 1, ..., totalPages]
} else {
  // Middle: show first, ..., window around current, ..., last
  pages = [1, '...', currentPage - 1, currentPage, currentPage + 1, '...', totalPages]
}
```

**Markup outline:**

```html
<nav aria-label="Page navigation" class="tw-flex tw-justify-center tw-items-center tw-gap-2 tw-font-oswald tw-mt-brick9 md:tw-mt-brick35 lg:tw-mt-brick50">
  <button :disabled="currentPage === 1" aria-label="Previous page" @click="$emit('update:page', currentPage - 1)">
    &lt;
  </button>

  <template v-for="(page, index) in visiblePages" :key="`${page}-${index}`">
    <span v-if="page === '...'" class="tw-px-2">...</span>
    <button v-else
      :class="page === currentPage ? 'tw-bg-brickBabsonGreen tw-text-white' : 'tw-text-brickSummerNight hover:tw-underline'"
      :aria-current="page === currentPage ? 'page' : undefined"
      @click="$emit('update:page', page)"
    >
      {{ page }}
    </button>
  </template>

  <button :disabled="currentPage === totalPages" aria-label="Next page" @click="$emit('update:page', currentPage + 1)">
    &gt;
  </button>
</nav>
```
