## Implementation Details

Write unit tests covering the `LocationExplorer.vue` component's rendering, interaction, and edge cases.

### LocationExplorer.vue Tests

Create `src/components/__tests__/LocationExplorer.spec.ts`:

**Test fixtures:**

```typescript
import { mount } from '@vue/test-utils'
import LocationExplorer from '../LocationExplorer.vue'

  { id: 'loc-1', name: 'Class Walk of 2019', mapImageUrl: 'https://example.com/map1.jpg' },

  { id: 'loc-3', name: '14-15-16', mapImageUrl: 'https://example.com/map3.jpg' },
]

const emptyLocations: ParkLocation[] = []

const locationWithNoMap: ParkLocation[] = [
  { id: 'loc-4', name: 'New Zone', mapImageUrl: '' },

```

**Rendering:**

- Renders a dark backdrop overlay.
- Renders the sidebar with all location names from the `locations` prop.
- Renders the close button (`×`) with `aria-label`.
- Renders the map image for the default-selected (first) location.
- Sidebar nav has `aria-label="Park locations"`.
- Applies the desktop/mobile location list typography contract: Oswald, black, `16px`, centered text, `0.08em` letter spacing, `24px` line height.

**Default selection:**

- On mount, the first location in the list is selected.
- The first location's map image URL is used as the `<img>` `src`.
- The first location's list item has the active/highlight class (e.g., `tw-font-bold`).

**Clicking a location:**

- Clicking a different location highlights it in the sidebar.
- Clicking a different location swaps the map image `src` to the new location's `mapImageUrl`.
- The previously selected location loses the highlight class.
- The component renders only one map `<img>` at a time; changing selection replaces that image rather than revealing a pre-rendered hidden image.

**Close behavior:**

- Clicking the `×` button emits `close`.
- Clicking the backdrop (`.self` on the outer container) emits `close`.
- Pressing `Escape` emits `close`.

**Empty locations:**

- When `locations` is an empty array, no list items render.
- No map image renders (fallback text shown or empty state).
- No errors on mount.

**No map image:**

- When the selected location has an empty `mapImageUrl`, the fallback text ("No map available") renders instead of an `<img>`.

**Mobile chevrons:**

- Chevron controls expose accessible names (`Scroll locations up` / `Scroll locations down`).
- Chevron visibility is driven by the scroll position of the list container itself.
- Clicking a chevron scrolls the list in that direction.

**Body scroll lock:**

- On mount, `document.body.style.overflow` is set to `'hidden'`.
- On unmount, `document.body.style.overflow` is restored to `''`.
