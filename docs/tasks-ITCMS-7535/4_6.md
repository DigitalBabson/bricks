## Implementation Details

Write Playwright E2E tests to verify location filtering, pill behavior, and "Clear all" against the running dev server.

### Test Scenarios

Create or update a spec file in `tests/e2e/`:

**Location list rendering:**

- Location list is visible inside the hero search form.
- Location list contains multiple items (populated from the API).
- List is scrollable if items exceed the visible area.

**Location-only filtering:**

- Click one or more locations in the list → bricks update to show bricks from the selected zone set.
- Selected locations are visually highlighted in the list.
- Pagination resets to page 1 when a location is selected.
- `totalPages` updates to reflect the filtered count.
- A location filter pill appears: "Brick Location: {name}".

**Location deselection:**

- Click the selected location again → filter is removed, default browse restores.
- Alternatively, click the `×` on the location pill → same behavior.

**Clear All:**

- Activate a location filter → pill appears.
- Click "Clear all" → location pill disappears, brick grid returns to default browse at page 1.

**Location filter + pagination:**

- Select a location → navigate to page 2 → bricks load for that location's page 2.
- Change to a different location → page resets to 1 with new location's bricks.

**Location filter resets pagination (from Task 2.6 follow-up):**

- Navigate to page 3 in default browse.
- Select a location → page resets to 1.

**Empty results:**

- Select a location that has no bricks (if one exists in dev data) → "No bricks match your criteria" message shows, pagination hides.

**Keyboard navigation:**

- Tab from the keyword field into the location listbox → listbox receives visible focus ring.
- Press `ArrowDown` → active option moves to the next item; the active item has a visible indicator.
- Press `ArrowUp` → active option moves to the previous item.
- Press `Home` → active option jumps to the first item.
- Press `End` → active option jumps to the last item.
- Press `Enter` on an active option → location is selected, bricks filter, pill appears.
- Press `Enter` on the already-selected option → location is deselected, default browse restores.
- Arrow past the visible area → list scrolls to keep the active option in view.

### Implementation Notes

- Use `page.locator('[role="listbox"]')` to target the location list.
- Use `page.locator('[role="option"]')` to target individual location items.
- Wait for network responses after filter changes: `page.waitForResponse(url => url.includes('bricks'))`.
- Pill assertions: check for text content like "Brick Location:" within the filter form area.
- For the "Clear all" test, verify both the pill disappearance and the brick grid content change.
