# Task 6.2 — Add BrickCard Hover States for Image Area + Location CTA

**Phase:** 6 — BrickCard Enhancements
**Spec:** [§3.9.1 Hover States (Mouse)](../03-features.md#391-hover-states-mouse), [§3.4 "Coming Soon" Bricks — Show Inscription](../03-features.md#34-coming-soon-bricks--show-inscription)
**Depends on:** Task 6.1

## Implementation Details

Implement the hover-state behavior from the `hovers.png` wireframe. The card should gain a semi-transparent white hover wash, the image area needs an "ENLARGE BRICK" affordance for real images, and the bottom action continues to behave as the location-details CTA.

**Current state:** The image element in `BrickCard.vue` has a simple opacity hover, and the bottom button already changes background on hover. There is no explicit image-action label, no semi-transparent white hover wash from the wireframe, and no differentiation between standard-image bricks and "Coming Soon" bricks.

### Card hover wash + image-area affordance

For bricks that are **not** `isComingSoon`:

- Add `tw-group` to the card wrapper so hover and `focus-within` can drive a shared visual state.
- Add a semi-transparent white overlay/wash in the image area that appears on card hover.
- Add a top-right overlay label:
  - Copy: `ENLARGE BRICK`
  - Include magnifying-glass icon/text treatment
  - White text on dark semi-transparent background
  - Hidden by default
  - Visible on `group-hover`
  - Also visible on `group-focus-within` so keyboard users get the same cue before the second tab stop is focused

Suggested structure:

```html
<article class="tw-group ...">
  <div class="brick-card__media tw-relative">
    <img ... />

    <div
      v-if="!isComingSoon"
      class="
        tw-absolute tw-inset-0 tw-bg-white/40
        tw-opacity-0 group-hover:tw-opacity-100 group-focus-within:tw-opacity-100
        tw-transition-opacity tw-duration-200
      "
    ></div>

    <div
      v-if="!isComingSoon"
      class="
        tw-absolute tw-top-2 tw-right-2
        tw-bg-black/70 tw-text-white tw-font-oswald tw-uppercase
        tw-px-3 tw-py-1 tw-text-sm
        tw-opacity-0 group-hover:tw-opacity-100 group-focus-within:tw-opacity-100
        tw-transition-opacity tw-duration-200
      "
    >
      Enlarge Brick 🔍
    </div>
  </div>
</article>
```

### Location CTA hover behavior

The bottom CTA remains the location-details action:

- Copy: `VIEW LOCATION DETAILS`
- Hover behavior: `tw-bg-brickMediumGreen tw-text-white`
- Text styling should stay aligned with the rest of the card typography

### Coming Soon variant

For `isComingSoon === true`:

- Do **not** render the enlarge overlay.
- Do **not** apply the normal image hover opacity treatment.
- Do **not** render the semi-transparent white hover wash over the placeholder image.
- Keep the "VIEW LOCATION DETAILS" button hover state unchanged.
- The image area remains non-interactive for mouse users.

**Files to modify:**

- `src/components/BrickCard.vue`
