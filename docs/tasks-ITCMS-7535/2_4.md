# Task 2.4 — Responsive Brick Grid (2-Column Mobile / 4-Column Desktop)

**Phase:** 2 — Sorting + Pagination
**Spec:** [§3.2.3 Responsive Brick Grid](../03-features.md#323-responsive-brick-grid-from-xd-wireframes)

---

## Implementation Details

Update the brick grid in `TheBricks.vue` to render 2 columns on mobile and 4 columns on desktop, matching the XD wireframes.

**Current state:** The grid uses `tw-grid-cols-1 md:tw-grid-cols-2 lg:tw-grid-cols-4`, which shows 1 column on small screens, 2 on medium, and 4 on large.

**Target state (aligned to current implementation breakpoint):**

- **Below `lg` (<1024px):** 2 columns — each card spans roughly half the viewport width.
- **Desktop (`lg` / ≥1024px):** 4 columns.

**Changes:**

Update the grid classes on the bricks container `<div>`:

```html
<!-- Old -->
<div class="... tw-grid tw-grid-cols-1 md:tw-grid-cols-2 lg:tw-grid-cols-4 ...">

<!-- New -->
<div class="... tw-grid tw-grid-cols-2 lg:tw-grid-cols-4 ...">
```

The change is minimal: replace `tw-grid-cols-1 md:tw-grid-cols-2 lg:tw-grid-cols-4` with `tw-grid-cols-2 lg:tw-grid-cols-4`.

**Key points:**

- The default (mobile-first) is 2 columns, not 1.
- The 4-column switch stays at the current `lg` breakpoint (`≥1024px`).
- The `md`-specific 2-column override is no longer needed because 2 columns are now the default below `lg`.
- Gap spacing (`tw-gap-brick3 md:tw-gap-brick5 lg:tw-gap-13`) may need review — ensure consistent gutters at 2-column width. The `lg:tw-gap-13` override can remain for wider screens.
- BrickCard images should still scale responsively within the smaller 2-column cells. Verify that thumbnails don't crop awkwardly on narrow viewports (~360px width → ~160px per card).

**Files to modify:**

- `src/components/TheBricks.vue` — update grid classes

---

## Acceptance Criteria

- [ ] Viewports below `lg` (<1024px) render brick cards in a 2-column grid.
- [ ] Desktop viewport (`lg` / ≥1024px) renders brick cards in a 4-column grid.
- [ ] The `tw-grid-cols-1` class is removed (no single-column layout).
- [ ] Gutter spacing looks consistent on small viewports (~360px).
- [ ] Brick card images scale correctly within the narrower 2-column cells.
- [ ] No horizontal overflow on mobile.
