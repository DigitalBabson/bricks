## Implementation Details

Add or expand Vitest coverage for the final BrickCard interaction model: placeholder overlay, hover/focus affordances, keyboard actions, mouse-click behavior, and accessible markup.

**Current test state:** `src/components/__tests__/BrickCard.spec.ts` already covers basic rendering, modals, and data-loading behavior. It does not yet fully verify the wireframe interaction contract for the "Coming Soon" overlay, first/second tab stops, or the dedicated enlarge button.

### Test coverage to add

Extend `src/components/__tests__/BrickCard.spec.ts` with tests for:

1. **Coming Soon overlay**
   - When `brick.brickImage === 'default'`, render the inscription overlay.
   - "Image Coming Soon" text is visible.
   - The enlarge overlay/button is not rendered.

2. **Normal-image variant**
   - Normal-image bricks do not show the "Image Coming Soon" text.
   - The enlarge button/overlay exists in the DOM.

3. **First tab stop / wrapper action**
   - The card wrapper has `tabindex="0"`.
   - The wrapper has `role="button"`.
   - The wrapper has a non-empty `aria-label`.
   - `keydown.enter` on the wrapper opens the location modal.
   - `keydown.space` on the wrapper opens the location modal.
   - Mouse click on the wrapper itself does **not** replace the explicit image/button mouse actions.

4. **Second tab stop / enlarge button**
   - The enlarge button exists only for non-placeholder bricks.
   - The enlarge button has an `aria-label`.
   - Clicking the enlarge button opens the image modal.
   - Keyboard activation on the enlarge button opens the image modal.
   - The enlarge button interaction does not also open location details.
   - Clicking the real-image media area opens the image modal.
   - `focus-within` state exposes the enlarge affordance even without mouse hover.

5. **Bottom CTA**
   - The visible `VIEW LOCATION DETAILS` button has `tabindex="-1"`.
   - Clicking the bottom button opens location details.

6. **Coming Soon keyboard contract**
   - Placeholder bricks expose only the wrapper tab stop.
   - No enlarge button is rendered or focusable.
   - Clicking the placeholder image area does not open the image modal.

7. **Shared location map style usage**
   - If the shared `ParkLocation` bootstrap shape changes in Phase 6, update `App.spec.ts` to verify `mapImageUrl` resolves from Drupal `image_style_uri` rather than only the raw file URL.
