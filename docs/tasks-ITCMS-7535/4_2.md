## Implementation Details

Expand `BrickFilter.vue` from a keyword-only input into a two-section filter bar with keyword search and a scrollable location list.

**Current state:** `BrickFilter.vue` has a single text input (`inscription` v-model), a clear button, and emits `update:inscription`. No location awareness.

**Target state (from XD wireframe):**

```
┌──────────────────────────────────┐
│ Search by Brick Inscription      │
│ ┌──────────────────────────────┐ │
│ │ Sample keyword            ×  │ │
│ └──────────────────────────────┘ │
│                                  │
│ Brick Locations                  │
│ ┌──────────────────────────────┐ │
│ │ 14-15-16                   ▲ │ │
│ │ 15-16-17                     │ │
│ │ Class Walk of 2019           │ │
│ │ Class Walk of 2020         ▼ │ │
│ └──────────────────────────────┘ │
│                                  │
│ [pills + Clear all — Task 4.4]   │
└──────────────────────────────────┘
```

**New props:**

| Prop | Type | Description |
|---|---|---|
| `inscription` (v-model) | `String` | Current keyword search text |
| `locationId` (v-model) | `String` | Selected location ID (empty = all) |
| `locations` | `Array<ParkLocation>` | All available locations (from App.vue) |
| `resultCount` | `Number` | Total matching bricks (for results bar — wired in Task 4.4) |

**New emits:**

| Event | Payload | Description |
|---|---|---|
| `update:inscription` | `String` | Keyword changed |
| `update:locationId` | `String` | Location filter changed |
| `clearAll` | — | Reset all filters (wired in Task 4.4) |

**Section A — Keyword input (existing, minor updates):**

- Keep the existing input with `update:inscription` emit.
- Keep the `×` clear button.
- Add the label "Search by Brick Inscription" above (already present).

**Section B — Location list (new):**

- Label: "Brick Locations" (Oswald font, white text, uppercase, matching Section A label style).
- **Scrollable list box** — a `<ul>` (or `<div role="listbox">`) with `overflow-y: auto` and a fixed max-height (roughly 4–5 visible items, ~160px).
- Each item is a `<li>` (or `<div role="option">`) displaying the location name.
- **Single-select:** clicking a location sets it as active (emits `update:locationId` with the location's ID). Clicking the already-selected location deselects it (emits empty string).
- Active/selected item: visually highlighted — e.g., `tw-bg-brickLightGreen tw-font-bold` or similar brand-colored treatment.
- Inactive items: `hover:tw-bg-gray-100` for hover feedback.
- Font: Zilla Slab for list items (body text font).

**Accessibility & Keyboard Navigation (required):**

- List container: `role="listbox"`, `aria-label="Brick Locations"`, `tabindex="0"`.
- Each item: `role="option"`, `aria-selected="true/false"`, `id` attribute for `aria-activedescendant`.
- The listbox is focusable and has a visible focus ring (e.g., `focus:tw-ring-2 focus:tw-ring-brickSummerNight`).
- `aria-activedescendant` on the listbox points to the currently focused option's `id`.

**Keyboard behavior (on the focused listbox):**

| Key | Action |
|---|---|
| `ArrowDown` | Move active option to the next item; wrap to first if at end |
| `ArrowUp` | Move active option to the previous item; wrap to last if at start |
| `Home` | Move active option to the first item |
| `End` | Move active option to the last item |
| `Enter` / `Space` | Select or deselect the active option (toggle) |

- The active (focused) option is visually distinct from the selected option — e.g., a subtle outline or background tint so the user can see where the cursor is before pressing Enter.
- When the listbox receives focus, the active option defaults to the currently selected location (if any) or the first item.
- Scrolling: the active option should scroll into view when navigated via keyboard (`scrollIntoView({ block: 'nearest' })`).

**Markup outline:**

```html
<form class="bricks__search-form tw-w-full tw-max-w-[540px] ..." @submit.prevent>
  <!-- Section A: Keyword -->
  <div class="tw-mb-4">

      <input :value="inscription" @input="..." type="text" />
      <button type="button" @click="resetSearch">×</button>
    </div>
  </div>

  <!-- Section B: Locations -->
  <div>
    <label
      id="locations-label"
      class="tw-font-oswald tw-text-white tw-uppercase tw-text-sm tw-font-semibold tw-block tw-mb-2"
    >
      Brick Locations
    </label>
    <ul
      ref="locationListbox"
      role="listbox"
      aria-labelledby="locations-label"
      :aria-activedescendant="activeDescendantId"
      tabindex="0"

        tw-bg-white tw-max-h-[160px] tw-overflow-y-auto

      </li>
    </ul>
```

**Data:**

```typescript
data() {
  return {
    activeIndex: -1,  // index of the keyboard-focused option (-1 = none)
  };
},
```

**Computed:**

```typescript
activeDescendantId(): string | undefined {
  if (this.activeIndex >= 0 && this.activeIndex < this.locations.length) {
    return `location-option-${this.locations[this.activeIndex].id}`;
  }
  return undefined;
},
```

**Methods:**

```typescript
selectLocation(id: string) {
  // Toggle: clicking the selected location deselects it
  const newId = id === this.locationId ? '' : id;
  this.$emit('update:locationId', newId);
},
resetSearch() {
  this.$emit('update:inscription', '');
},

// Keyboard navigation
onListboxFocus() {
  if (this.activeIndex >= 0) return; // preserve existing position
  // Default to selected location or first item
  const selectedIdx = this.locations.findIndex(l => l.id === this.locationId);
  this.activeIndex = selectedIdx >= 0 ? selectedIdx : 0;
},
moveActive(delta: number) {
  if (this.locations.length === 0) return;
  let next = this.activeIndex + delta;
  if (next < 0) next = this.locations.length - 1;         // wrap to end
  if (next >= this.locations.length) next = 0;             // wrap to start
  this.activeIndex = next;
  this.scrollActiveIntoView();
},
moveActiveToFirst() {
  this.activeIndex = 0;
  this.scrollActiveIntoView();
},
moveActiveToLast() {
  this.activeIndex = this.locations.length - 1;
  this.scrollActiveIntoView();
},
toggleActiveSelection() {
  if (this.activeIndex < 0 || this.activeIndex >= this.locations.length) return;
  this.selectLocation(this.locations[this.activeIndex].id);
},
scrollActiveIntoView() {
  this.$nextTick(() => {
    const listbox = this.$refs.locationListbox as HTMLElement | undefined;
    const activeEl = listbox?.querySelector(`#${this.activeDescendantId}`);
    activeEl?.scrollIntoView({ block: 'nearest' });
  });
},
```

## Acceptance Criteria

- [ ] BrickFilter accepts `locationId`, `locations`, and `resultCount` props in addition to `inscription`.
- [ ] Emits `update:locationId` when a location is selected or deselected.
- [ ] Location list renders all items from the `locations` prop, sorted as received (already sorted by API).
- [ ] List is scrollable with a fixed max-height (~4–5 visible items).
- [ ] Selected location is visually highlighted.
- [ ] Clicking the selected location deselects it (emits empty string).
- [ ] "Brick Locations" label matches the style of "Search by Brick Inscription" label.
- [ ] Existing keyword input and clear button still work.
- [ ] Empty `locations` array renders no list items (graceful empty state).
- [ ] **ARIA:** `role="listbox"` on container, `role="option"` + `aria-selected` on items, `aria-activedescendant` tracks the active option.
- [ ] **Focus:** Listbox is focusable (`tabindex="0"`) with a visible focus ring.
- [ ] **Arrow Down / Arrow Up:** Moves the active option through the list with wrapping.
- [ ] **Home / End:** Jumps to the first / last option.
- [ ] **Enter / Space:** Selects or deselects the active option (toggle).
- [ ] Active option scrolls into view when navigated via keyboard.
- [ ] Active option has a distinct visual indicator (e.g., inset ring) separate from the selected highlight.
- [ ] On focus, active option defaults to the selected location (if any) or the first item.
