## Implementation Details

Expand `BrickFilter.vue` from a keyword-only input into a two-section filter bar with keyword search and a scrollable location list.

**Current state:** `BrickFilter.vue` has a single text input (`inscription` v-model) and emits `update:inscription`. No location awareness.

**Target state (from XD wireframe):**

```
┌──────────────────────────────────┐
│ Search by Brick Inscription      │
│ ┌──────────────────────────────┐ │
│ │ Sample keyword            ×  │ │
│ └──────────────────────────────┘ │
│                                  │
│ Brick Locations                  │
│ ┌──────────────────────────────┐ │
│ │ 14-15-16                   ▲ │ │
│ │ 15-16-17                     │ │
│ │ Class Walk of 2019           │ │
│ │ Class Walk of 2020         ▼ │ │
│ └──────────────────────────────┘ │
│                                  │
│ [pills + Clear all — Task 4.4]   │
└──────────────────────────────────┘
```

**New props:**

| Prop | Type | Description |
|---|---|---|
| `inscription` (v-model) | `String` | Current keyword search text |
| `locationIds` (v-model) | `String[]` | Selected location IDs (empty array = all) |
| `locations` | `Array<ParkLocation>` | All available locations (from App.vue) |

**New emits:**

| Event | Payload | Description |
|---|---|---|
| `update:inscription` | `String` | Keyword changed |
| `update:locationIds` | `String[]` | Location filter changed |
| `clearAll` | — | Reset all filters (wired in Task 4.4) |

**Section A — Keyword input (existing, minor updates):**

- Keep the existing input with `update:inscription` emit.
- Add the label "Search by Brick Inscription" above (already present) using Zilla Slab at `18px`, title case.
- Do **not** include a dedicated inline clear button inside the keyword field. The form-level inscription pill `×`, `Clear all`, or manual input clearing handle reset behavior.

**Section B — Location list (new):**

- Label: "Brick Locations" (Zilla Slab, white, `18px`, title case, matching Section A label style from the wireframe rather than forced uppercase).
- **Scrollable list box** — a `<ul>` (or `<div role="listbox">`) with `overflow-y: auto` and a max-height that shows at most three visible items on both mobile and desktop before scrolling. The listbox uses square corners.
- Each item is a `<li>` (or `<div role="option">`) displaying the location name.
- **Multi-select:** clicking a location toggles it in the active selection set (emits `update:locationIds` with the full selected-ID array).
- Active/selected item: visually highlighted — e.g., `tw-bg-brickLightGreen tw-font-bold` or similar brand-colored treatment.
- Inactive items: `hover:tw-bg-gray-100` for hover feedback.
- Font: Oswald at `16px` for list items.

**Accessibility & Keyboard Navigation (required):**

- List container: `role="listbox"`, `aria-label="Brick Locations"`, `aria-multiselectable="true"`, `tabindex="0"`.
- Each item: `role="option"`, `aria-selected="true/false"`, `id` attribute for `aria-activedescendant`.
- The listbox is focusable and has a visible focus ring (e.g., `focus:tw-ring-2 focus:tw-ring-brickSummerNight`).
- `aria-activedescendant` on the listbox points to the currently focused option's `id`.

**Keyboard behavior (on the focused listbox):**

| Key | Action |
|---|---|
| `ArrowDown` | Move active option to the next item; wrap to first if at end |
| `ArrowUp` | Move active option to the previous item; wrap to last if at start |
| `Home` | Move active option to the first item |
| `End` | Move active option to the last item |
| `Enter` / `Space` | Toggle the active option in the selected set |

- The active (focused) option is visually distinct from the selected option — e.g., a subtle outline or background tint so the user can see where the cursor is before pressing Enter.
- When the listbox receives focus, the active option defaults to the first selected location (if any) or the first item.
- Scrolling: the active option should scroll into view when navigated via keyboard (`scrollIntoView({ block: 'nearest' })`).

**Markup outline:**

```html
<form class="bricks__search-form tw-w-full tw-max-w-[540px] ..." @submit.prevent>
  <!-- Section A: Keyword -->
  <div class="tw-mb-4">
    <label ...>Search by Brick Inscription</label>
    <div class="tw-relative tw-w-full">
      <input :value="inscription" @input="..." type="text" />
    </div>
  </div>

  <!-- Section B: Locations -->
  <div>
    <label
      id="locations-label"
      class="tw-font-zilla tw-text-white tw-text-[18px] tw-font-semibold tw-leading-tight tw-block tw-mb-2"
    >
      Brick Locations
    </label>
    <ul
      ref="locationListbox"
      role="listbox"
      aria-labelledby="locations-label"
      :aria-activedescendant="activeDescendantId"
      tabindex="0"
      class="
        tw-bg-white tw-max-h-[108px] tw-overflow-y-auto
        tw-border tw-border-gray-300
        focus:tw-ring-2 focus:tw-ring-brickSummerNight focus:tw-outline-none

      @keydown.arrow-down.prevent="moveActive(1)"
      @keydown.arrow-up.prevent="moveActive(-1)"

        v-for="(loc, index) in locations"

        :class="[
          locationIds.includes(loc.id) ? 'tw-bg-brickLightGreen tw-font-bold' : '',

        class="tw-px-3 tw-py-2 tw-cursor-pointer tw-font-oswald tw-text-[16px] tw-leading-tight"
        @click="selectLocation(loc.id)"
      >
        {{ loc.name }}
      </li>
    </ul>
  </div>

  <!-- Section C: Pills + Clear All — added in Task 4.4 -->
```

**Data:**

```typescript
data() {
  return {
    activeIndex: -1,  // index of the keyboard-focused option (-1 = none)
  };
},
```

**Computed:**

```typescript
activeDescendantId(): string | undefined {
  if (this.activeIndex >= 0 && this.activeIndex < this.locations.length) {
    return `location-option-${this.locations[this.activeIndex].id}`;
  }
  return undefined;
},
```

**Methods:**

```typescript
selectLocation(id: string) {
  // Toggle: clicking the selected location deselects it
  const newIds = this.locationIds.includes(id)
    ? this.locationIds.filter(locationId => locationId !== id)
    : [...this.locationIds, id];
  this.$emit('update:locationIds', newIds);
},
// Keyboard navigation
onListboxFocus() {
  if (this.activeIndex >= 0) return; // preserve existing position
  // Default to selected location or first item
  const selectedIdx = this.locations.findIndex(l => this.locationIds.includes(l.id));
  this.activeIndex = selectedIdx >= 0 ? selectedIdx : 0;
},
moveActive(delta: number) {
  if (this.locations.length === 0) return;
  let next = this.activeIndex + delta;
  if (next < 0) next = this.locations.length - 1;         // wrap to end
  if (next >= this.locations.length) next = 0;             // wrap to start
  this.activeIndex = next;
  this.scrollActiveIntoView();
},
moveActiveToFirst() {
  this.activeIndex = 0;
  this.scrollActiveIntoView();
},
moveActiveToLast() {
  this.activeIndex = this.locations.length - 1;
  this.scrollActiveIntoView();
},
toggleActiveSelection() {
  if (this.activeIndex < 0 || this.activeIndex >= this.locations.length) return;
  this.selectLocation(this.locations[this.activeIndex].id);
},
scrollActiveIntoView() {
  this.$nextTick(() => {
    const listbox = this.$refs.locationListbox as HTMLElement | undefined;
    const activeEl = listbox?.querySelector(`#${this.activeDescendantId}`);
    activeEl?.scrollIntoView({ block: 'nearest' });
  });
},
```

## Acceptance Criteria

- [x] BrickFilter accepts `locationIds` and `locations` props in addition to `inscription`.
- [x] Emits `update:locationIds` when location selection changes.
- [x] Location list renders all items from the `locations` prop, sorted as received (already sorted by API).
- [x] List is scrollable with a max-height that shows at most three visible items on both mobile and desktop.
- [x] Selected location is visually highlighted.
- [x] Clicking the selected location deselects it (emits empty string).
- [x] "Brick Locations" label matches the style of "Search by Brick Inscription" label.
- [x] Existing keyword input still works without a dedicated inline clear button.
- [x] Empty `locations` array renders no list items (graceful empty state).
- [x] **ARIA:** `role="listbox"` on container, `role="option"` + `aria-selected` on items, `aria-activedescendant` tracks the active option.
- [x] **Focus:** Listbox is focusable (`tabindex="0"`) with a visible focus ring.
- [x] **Arrow Down / Arrow Up:** Moves the active option through the list with wrapping.
- [x] **Home / End:** Jumps to the first / last option.
- [x] **Enter / Space:** Selects or deselects the active option (toggle).
- [x] Active option scrolls into view when navigated via keyboard.
- [x] Active option has a distinct visual indicator (e.g., inset ring) separate from the selected highlight.
- [x] On focus, active option defaults to the selected location (if any) or the first item.

## Findings

- The listbox keyboard model was implemented as required behavior: focus ring, `aria-activedescendant`, wrapping arrow navigation, Home/End, Enter/Space toggle, and active-option scrolling.
- `BrickFilter.vue` now supports the full Phase 4 prop/emit surface so the location list, pills, and clear-all flow can be wired without another contract change.
- The wireframe PNG was used as the source of truth for searchbox typography, so the labels are rendered in title case rather than forced uppercase.
- The wireframe PNG was also used as the source of truth for typography sizes: labels are `18px` Zilla Slab and location options are `16px` Oswald.
- The dedicated inline keyword clear button was intentionally removed because the form-level filter actions already provide reset behavior.
- The location listbox now uses a three-row visible window on both mobile and desktop, with square corners to match the updated treatment.
- The final design does not include a result-count display in `BrickFilter`, so the filter component stays focused on inputs and reset actions only.
- Verified after implementation with `npm run lint` and `npm run typecheck`.
