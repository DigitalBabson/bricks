## Implementation Details

Replace the append-based "load more" pattern in `TheBricks.vue` with page-based state management for Drupal-backed browse results, using the Drupal response's `meta.count` to calculate total pages.

**Current state:**

```typescript
data() {
  return {
    bricks: [] as Brick[],
    offset: 0,
    nextPage: false,    // boolean from links.next
    showMessage: false
  };
}
```

- `loadMore()` calls `fetchBricks(inscription, offset + 20)` and **appends** results.
- Pagination visibility is based on `nextPage` boolean (presence of `links.next`).

**New state:**

```typescript
data() {
  return {
    bricks: [] as Brick[],
    currentPage: 1,
    totalPages: 1,
    pageSize: 20,
    showMessage: false
  };
}
```

**Changes:**

1. **Replace `offset` / `nextPage` with `currentPage` / `totalPages` / `pageSize`.**

2. **Read `meta.count` from Drupal response.** The `/jsonapi/bricks` endpoint returns `meta.count` (total number of matching bricks). Use it to calculate `totalPages` for default browse mode, and reuse the same pattern for Drupal-backed park-location filtering in Phase 4:

   ```typescript
   const totalCount = response.data.meta?.count ?? 0;
   this.totalPages = Math.ceil(totalCount / this.pageSize);
   ```

3. **Update `BrickApiResponse` type** in `src/types/index.ts` to include `meta`:

   ```typescript
   export interface BrickApiResponse {
     data: BrickApiItem[]
     links: { next?: { href: string } }
     meta?: { count?: number }
   }
   ```

4. **Calculate `page[offset]` from `currentPage`:**

   ```typescript
   const offset = (this.currentPage - 1) * this.pageSize;
   ```

5. **Replace bricks array on page change** (not append):

   ```typescript
   // Always replace, never append
   this.bricks = data;
   ```

6. **Handle page changes from Pagination:** Replace `loadMore()` with a `goToPage(page: number)` method:

   ```typescript
   goToPage(page: number) {
     this.currentPage = page;
     this.fetchBricks(this.inscription);
   }
   ```

7. **Scroll to the top of the brick grid on page change.** After fetching completes, scroll the grid container into view using a template ref on the grid element:

   ```typescript
   // template: <div ref="brickGrid" class="...">
   this.$nextTick(() => {
     this.$refs.brickGrid?.scrollIntoView({ behavior: 'smooth', block: 'start' });
   });
   ```

8. **Reset to page 1 when the active Drupal query changes.** In Phase 2, that mainly means resetting on browse-state changes such as a new page source or any temporary local filter hooks. In Phase 4, apply the same reset behavior to `locationId`. Searchstax keyword search is handled in Phase 3.

9. **Update template** to use new Pagination API:

   ```html
   <!-- Old -->
   <pagination v-if="nextPage" @loadmore="loadMore" />

   <!-- New -->
   <pagination
     v-if="totalPages > 1"
     :currentPage="currentPage"
     :totalPages="totalPages"
     @update:page="goToPage"
   />
   ```

**Files to modify:**

- `src/components/TheBricks.vue` — rewrite data, methods, template
- `src/types/index.ts` — add `meta` to `BrickApiResponse`
