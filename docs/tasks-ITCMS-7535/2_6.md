## Implementation Details

Write Playwright E2E tests to verify the sorting, page navigation, and responsive grid behavior against the running dev server.

### Test Scenarios

Create or update a spec file in `tests/e2e/`:

**Default browse — alphabetical order:**

- Page loads with bricks visible.
- First brick inscription is alphabetically before the last brick inscription on the page (verify A–Z ordering).

**Page navigation:**

- Pagination control is visible at the bottom of the grid.
- Page 1 is highlighted/active on initial load.
- Previous arrow (`<`) is disabled on page 1.
- Click page 2 → new bricks load (different from page 1), page 2 is now active.
- Previous arrow is now enabled.
- Viewport scrolls to top of the brick grid after page change.
- Click `>` (next) → advances to page 3.
- Navigate to the last page → next arrow (`>`) is disabled.

**Ellipsis behavior (visual check):**

- On page 1 with many pages, ellipsis and last page number are visible.
- After navigating to a middle page, both leading and trailing ellipsis appear.

**Location filter resets pagination (Phase 4 follow-up):**

- After Phase 4 lands, navigate to page 3.
- Select a park location filter.
- Pagination resets to page 1.
- Results reflect the selected location.
- Clear the location filter → pagination returns to the full set at page 1.

**Responsive grid — mobile (viewport 375×667):**

- Brick grid renders in 2 columns.
- Cards are approximately half the viewport width.
- No horizontal scrollbar.

**Responsive grid — desktop (viewport 1280×800):**

- Brick grid renders in 4 columns.

### Implementation Notes

- Use `page.waitForResponse()` or `page.waitForSelector()` to handle async data loading.
- For scroll-to-top verification, check `window.scrollY` or element bounding rect after page change.
- Mobile tests: use `page.setViewportSize({ width: 375, height: 667 })` (or Playwright's device emulation).
- Alphabetical check: extract the visible inscriptions and compare the list to a sorted copy rather than checking only the first and last cards.

**Files to create/modify:**

- `tests/e2e/pagination.spec.ts` — new
- `tests/e2e/responsive-grid.spec.ts` — new (or add scenarios to an existing layout spec)
