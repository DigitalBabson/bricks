## Implementation Details

Create the `LocationExplorer.vue` overlay component with the desktop two-column layout: a left sidebar listing all park zones and a right panel displaying the selected zone's map image. Mobile responsiveness is handled separately in Task 5.3.

**Current state:** No location-browsing overlay exists. Location maps are only visible per-brick inside `BrickCard.vue` modals. `App.vue` already has `showLocationExplorer: boolean` in data and, after Phase 4, `locations: ParkLocation[]` populated by `fetchLocations()` using the `field_brick_zone_image` relationship.

**Data-loading strategy:** Reuse the shared `locations: ParkLocation[]` dataset fetched once at `App.vue` level from `{DEV_DRUPAL_ENDPOINT}/parkLocations?include=field_brick_zone_image,field_brick_zone_image.field_media_image&fields[parkLocation]=name,field_brick_zone_image&fields[media--image]=field_media_image&fields[file--file]=uri,url`. That request should provide all location names plus the related media/file records needed to resolve map image URLs up front, but the overlay must not preload/render every map image. Only the currently selected location's `<img>` should be mounted so the browser fetches maps on demand as the user clicks through the list.

**Endpoint scope:** This Phase 5 change is limited to the shared park-locations fetch used by the overlay/filter. Do not switch the bricks listing/search/hydration endpoints here; that cutover belongs to Phase 3.

**Props:**

| Prop | Type | Description |
|---|---|---|
| `locations` | `ParkLocation[]` | All park locations (id, name, mapImageUrl) from App.vue |

**Emits:** `@close`

**State:**

```typescript
data() {
  return {
    selectedZoneId: '',  // set to first location's ID on mount/watch
  }
},
```

**Default selection:** `selectedZoneId` defaults to the first location in the sorted list (`this.locations[0]?.id ?? ''`). Set this in a watcher on `locations` (in case the prop is populated after mount) and also in `mounted()`.

**Image-loading behavior:** The right panel should render exactly one map image at a time: the currently selected location's `mapImageUrl`. Do not render hidden `<img>` tags for every location and do not add `<link rel="preload">` entries for all map images.

**Computed:**

```typescript
selectedLocation(): ParkLocation | undefined {
  return this.locations.find(loc => loc.id === this.selectedZoneId);
},
```

**ESC key to close:**

```typescript
mounted() {
  this.selectedZoneId = this.locations[0]?.id ?? ''
  document.addEventListener('keydown', this.onKeydown)
},
beforeUnmount() {
  document.removeEventListener('keydown', this.onKeydown)
},
methods: {
  onKeydown(e: KeyboardEvent) {
    if (e.key === 'Escape') {
      this.$emit('close')
    }
  },
},
```

**Body scroll lock:** When the overlay is mounted, prevent background scrolling by adding `overflow: hidden` to `<body>`. Restore on unmount:

```typescript
mounted() {
  document.body.style.overflow = 'hidden'
},
beforeUnmount() {
  document.body.style.overflow = ''
},
```

**Transition:** Wrap the teleported content in `<Transition name="fade">` for consistency with the existing `UiModal` pattern.
