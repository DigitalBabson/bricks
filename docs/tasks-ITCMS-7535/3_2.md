## Implementation Details

Create a standalone service module that handles all SearchStax API communication. This service is a pure async function — no Vue dependencies, no debouncing (debounce lives in the caller). It accepts search parameters and returns `Brick[]` objects constructed directly from SearchStax fields (zero-hydration), plus `numFound` for pagination.

**Current state:** No SearchStax service exists. Keyword search currently uses Drupal `CONTAINS` filter in `TheBricks.vue buildUrl()`.

**Building the request URL:**

The SearchStax `emselect` endpoint uses query parameters:

```
GET {endpoint}
  ?q=*:*
  &fq=tcngramm_X3b_en_description:{keyword}
  &rows={pageSize}
  &start={offset}
  &fl=ss_uuid,ss_zone_uuid,ss_file_img_uuid,tcngramm_X3b_en_description
  &wt=json
```

**Combined keyword + location:** When `locationIds` has entries, add one `fq` parameter per location UUID using an OR group:

```
&fq=ss_zone_uuid:({uuid1} OR {uuid2} OR {uuid3})
```

This filters results to bricks in any of the selected zones while matching the keyword. The `ss_zone_uuid` field stores the Drupal park location UUID, which maps directly to the `locationIds` array from BrickFilter.

**Note on `fl` (field list):** Request only the four fields needed to construct a `Brick` object: `ss_uuid`, `ss_zone_uuid`, `ss_file_img_uuid`, and `tcngramm_X3b_en_description`. This enables zero-hydration — no Drupal batch call is needed.

**Headers:**

```
Authorization: Token {token}
```

**Parsing the response — construct `Brick` objects directly:**

```typescript
const data = response.data as SearchstaxResponse
return {
  bricks: data.response.docs.map(doc => ({
    id: doc.ss_uuid,
    inscription: doc['tcngramm_X3b_en_description'][0] ?? '',
    brickImage: doc.ss_file_img_uuid,
    brickParkLocation: doc.ss_zone_uuid,
  })),
  numFound: data.response.numFound,
}
```

**Field mapping:**

| `Brick` property | SearchStax field | Notes |
|---|---|---|
| `id` | `ss_uuid` | Brick media entity UUID |
| `inscription` | `tcngramm_X3b_en_description[0]` | First element of array |
| `brickImage` | `ss_file_img_uuid` | File entity UUID for image |
| `brickParkLocation` | `ss_zone_uuid` | Park location UUID |

**Error handling:**

The function should throw on network errors or non-200 responses. The caller (TheBricks) handles the fallback to Drupal CONTAINS search (Task 3.5).
