## Implementation Details

Adapt `LocationExplorer.vue` from the desktop two-column layout to a single-column stacked layout on mobile viewports, matching the XD mobile wireframe.

**Desktop layout (from Task 5.1):** Left sidebar + right map panel, side by side.

**Mobile layout (from XD mobile wireframe):**

```
┌──────────────────────────────┐
│                        [ × ] │
│  ┌──────────────────────┐    │
│  │                      │    │
│  │  [Selected zone's    │    │
│  │   map image,         │    │
│  │   full-width]        │    │
│  │                      │    │
│  └──────────────────────┘    │
│                              │
│           ∧                  │
│   ┌──────────────────┐       │
│   │ Class Walk 2019  │       │
│   │ Class Walk 2020  │       │
│   │ Class Walk 2021  │       │
│   │ Rodger Babson    │       │
│   └──────────────────┘       │
│           ∨                  │
│                              │
└──────────────────────────────┘
```

- **Map on top:** full-width image, no sidebar.
- **Location list below:** vertically scrollable, horizontally centered. The list container itself is the only scrollable region.
- **Chevron scroll controls:** Up (`∧`) and down (`∨`) arrows at the top and bottom of the list area. They indicate scrollability and also act as tappable/clickable controls that scroll the list in that direction.
- **Selected location:** medium weight text with underline, matching the desktop highlight.
- **Dark/black background** fills the full viewport at `0.87` opacity.
- **Close button (`×`):** top-right corner, above the map.
- **Image loading:** still render only the selected location's map image. Mobile responsiveness does not change the no-preload strategy.

**Breakpoint strategy:**

Use Tailwind responsive classes to switch between layouts. The `md` breakpoint (768px) is a good threshold:

- Below `md`: stacked mobile layout.
- `md` and above: side-by-side desktop layout.

**Template changes to LocationExplorer.vue:**

Replace the fixed two-column `tw-flex` with responsive classes:


**Chevron scroll indicator logic:**

```typescript
data() {
  return {
    selectedZoneId: '',
    showUpChevron: false,
    showDownChevron: false,
  }
},
methods: {
  updateChevrons() {
    const list = this.$refs.locationList as HTMLElement | undefined
    if (!list) return
    this.showUpChevron = list.scrollTop > 0
    this.showDownChevron = list.scrollTop + list.clientHeight < list.scrollHeight - 1
  },
  scrollLocations(direction: 'up' | 'down') {
    const list = this.$refs.locationList as HTMLElement | undefined
    if (!list) return
    const amount = Math.max(44, Math.round(list.clientHeight * 0.35))
    list.scrollBy({
      top: direction === 'up' ? -amount : amount,
      behavior: 'smooth',
    })
  },
},
mounted() {
  this.$nextTick(() => this.updateChevrons())
},
```

Also call `updateChevrons()` on window resize and when `locations` changes (via watcher).

**Mobile overlay container adjustments:**

On mobile, the overlay should fill the full viewport rather than using the constrained desktop dimensions:

```html
<div
  class="
    tw-relative tw-shadow-xl
    tw-w-full tw-h-full
    md:tw-w-[90vw] md:tw-max-w-[1100px] md:tw-h-[80vh]
    md:tw-rounded-lg
    tw-flex tw-flex-col
  "
>
```

On mobile the overlay takes `tw-w-full tw-h-full` (full viewport), while on desktop it uses the contained card dimensions with rounded corners. The black `0.87` opacity backdrop remains visible behind the white `0.87` opacity list panel and the map content on both breakpoints.

**Mobile map sizing:**

The map on mobile should be constrained to roughly 40–50% of the viewport height so the location list remains visible:

```html
<div class="
  tw-order-1 md:tw-order-2
  tw-flex-shrink-0 md:tw-flex-shrink md:tw-flex-1
  tw-h-[40vh] md:tw-h-auto
  tw-flex tw-items-center tw-justify-center tw-p-4 tw-overflow-auto
">
```

## Acceptance Criteria

- [ ] Below `md` breakpoint: map displays on top, location list displays below.
- [ ] Above `md` breakpoint: sidebar on left, map on right (unchanged from Task 5.1).
- [ ] Mobile overlay fills the full viewport (no rounded corners, no constrained width).
- [ ] Location list container is the only vertically scrollable region on mobile.
- [ ] Up chevron control shows when the list is scrolled down from the top.
- [ ] Down chevron control shows when more items are below the visible area.
- [ ] Chevrons are hidden on desktop.
- [ ] Selected location in mobile list uses the resolved medium/underline state.
- [ ] Location list items are horizontally centered on mobile and desktop.
- [ ] Map image is constrained to ~40–50% viewport height on mobile so the list remains visible.
- [ ] Mobile layout still renders only the selected location's image instead of mounting hidden images for all locations.
- [ ] Close button remains accessible in the top-right corner on mobile.
- [ ] Tapping/clicking a chevron scrolls the list in that direction.
- [ ] Chevron controls expose accessible names and are keyboard reachable.
- [ ] No horizontal overflow on any viewport width.
