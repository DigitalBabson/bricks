## Implementation Details

Update `TheBricks.vue` to route keyword searches through SearchStax (zero-hydration — no Drupal batch call needed), while keeping the existing Drupal paths for default browse and location-only filtering.

**Current state:** All fetch paths in TheBricks go through Drupal JSON:API via `buildUrl()`. Keyword search uses `filter[brickInscription][operator]=CONTAINS`.

**Target routing rule:**

| Condition | Data path |
|---|---|
| No keyword, no location | Drupal `/bricks?sort=brickInscription&page[limit]=20` (unchanged) |
| No keyword, location(s) selected | Drupal `/bricks?filter[brickParkLocation.id][operator]=IN` (unchanged) |
| Keyword ≥ 3 chars, no location | **SearchStax** keyword search → `Brick[]` constructed directly |
| Keyword ≥ 3 chars + location(s) | **SearchStax** keyword + location search → `Brick[]` constructed directly |

**Changes to `TheBricks.vue`:**

1. **Inject SearchStax config:**

```typescript
import { searchstaxEndpointKey, searchstaxTokenKey } from '../types/index'

inject: {
  defaultEnv: { from: defaultEnvKey, default: '' },
  defaultUrl: { from: defaultUrlKey, default: '' },
  searchstaxEndpoint: { from: searchstaxEndpointKey, default: '' },
  searchstaxToken: { from: searchstaxTokenKey, default: '' },
},
```

2. **Import the service:**

```typescript
import { searchBricks } from '../services/searchstax'
```

3. **Add a SearchStax fetch method:**

```typescript
async fetchViaSearchstax() {
  const offset = (this.currentPage - 1) * this.pageSize;

  // Single SearchStax call — returns fully constructed Brick[] (zero-hydration)
  const result = await searchBricks({
    endpoint: this.searchstaxEndpoint as string,
    token: this.searchstaxToken as string,
    keyword: this.inscription,
    locationIds: this.locationIds.length > 0 ? this.locationIds : undefined,
    pageSize: this.pageSize,
    offset,
  });

  this.bricks = result.bricks;
  this.showMessage = result.bricks.length === 0;

  // Use SearchStax numFound for pagination (not Drupal meta.count)
  this.totalPages = Math.ceil(result.numFound / this.pageSize) || 1;
  this.$emit('update:totalCount', result.numFound);
},
```

4. **Update `fetchBricks()` to route based on keyword presence:**

```typescript
async fetchBricks() {
  try {
    const hasKeyword = this.inscription.length >= 3;

    if (hasKeyword) {
      await this.fetchViaSearchstax();
    } else {
      // Existing Drupal path (default browse or location-only filter)
      const offset = (this.currentPage - 1) * this.pageSize;
      const url = this.buildUrl(offset);
      const response = await axios.get<BrickApiResponse>(url);
      // ... (existing parsing logic, unchanged)
    }

    // Scroll grid to top on page change
    if (this.currentPage > 1) {
      this.$nextTick(() => {
        const grid = this.$refs.brickGrid as HTMLElement | undefined;
        grid?.scrollIntoView?.({ behavior: 'smooth', block: 'start' });
      });
    }
  } catch {
    this.bricks = [];
    this.totalPages = 1;
    this.showMessage = this.locationIds.length > 0 || this.inscription.length >= 3;
    this.$emit('update:totalCount', 0);
  }
},
```

5. **Simplify `buildUrl()`** — remove the keyword CONTAINS branch since that path now goes through SearchStax:

```typescript
buildUrl(offset: number): string {
  if (this.locationIds.length > 0) {
    const encodedIds = this.locationIds.map((id) => encodeURIComponent(id)).join(',');
    return this.apiUrl +
      `bricks?page[limit]=${this.pageSize}` +
      `&sort=brickInscription` +
      `&filter[brickParkLocation.id][operator]=IN` +
      `&filter[brickParkLocation.id][value]=${encodedIds}` +
      `&page[offset]=${offset}`;
  }

  // Default alphabetical browse
  return this.apiUrl +
    `bricks?page[limit]=${this.pageSize}` +
    `&page[offset]=${offset}` +
    `&sort=brickInscription`;
},
```

6. **Update the `inscription` + `locationIds` watchers for combined search:**

The `inscription` watcher (already debounced in Task 3.3) handles keyword-only and combined paths. The `locationIds` watcher already works correctly because `fetchBricks()` checks `this.inscription.length >= 3` to decide the path. No change needed to the watcher itself, but it's called out here for clarity.
