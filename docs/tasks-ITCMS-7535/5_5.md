## Implementation Details

Write Playwright E2E tests to verify the LocationExplorer overlay works end-to-end against the running dev server, covering both desktop and mobile viewports.

### Test Scenarios

Create `tests/e2e/location-explorer.spec.ts`:

**Opening the overlay (desktop):**

- Click the "View Map of Brick Locations" trigger in the hero area → overlay appears.
- Overlay has a dark backdrop covering the viewport.
- Overlay contains a sidebar list and a map image panel.

**Opening the overlay (mobile):**

- On a mobile viewport (e.g., Playwright's `iPhone 12` or custom `375×812`), the floating bottom-right trigger is visible.
- Click the floating trigger → overlay appears.
- Overlay fills the full viewport on mobile.

**Location list rendering:**

- Sidebar lists multiple locations (populated from the API).
- Locations are in alphabetical order (matching API `sort=name`).
- The first location is selected by default (highlighted).
- The overlay uses the already-fetched shared location dataset; opening it should not require a second per-location bootstrap call.

**Selecting a location:**

- Click a different location in the sidebar → that location is highlighted.
- The map image updates to show the selected location's zone map.
- The previously selected location loses the highlight.

**Map image display:**

- The map image `src` corresponds to the selected location's `mapImageUrl`, resolved from the included `field_brick_zone_image -> field_media_image -> file--file` resources.
- The image has an `alt` attribute describing the location.
- The DOM contains the active map image only; the test should not assume all location maps are preloaded/rendered up front.

**Close overlay — close button:**

- Click the `×` button → overlay disappears.
- The page behind is interactive again.

**Close overlay — backdrop:**

- Click the dark backdrop outside the content area → overlay closes.

**Close overlay — Escape key:**

- Press `Escape` → overlay closes.

**Mobile layout:**

- On mobile viewport: map image is displayed above the location list (stacked layout).
- Location list items are horizontally centered.
- Selected location item uses the resolved medium/underline state.
- The list is scrollable if items exceed the visible area.

**Accessibility recommendations:**

- Verify the overlay exposes `role="dialog"` and `aria-modal="true"`.
- Verify focus moves into the overlay on open and returns to the triggering button on close.
- Verify chevrons are real buttons with accessible names and keyboard support.
- Keep the close button query based on its accessible name.

**Desktop layout:**

- On desktop viewport: sidebar is on the left, map is on the right (side-by-side).
- Sidebar is scrollable if locations exceed the visible height.

**Re-open persistence:**

- Open overlay → select a location → close → re-open → verify the overlay defaults back to the first location (fresh state on each open).

### Implementation Notes

- Use `page.locator('nav[aria-label="Park locations"]')` to target the sidebar.
- Wait for the overlay to appear after clicking the trigger: `page.locator('[aria-label="Close location explorer"]').waitFor()`.
- Wait for the map image to load: `page.locator('img[alt*="Map of"]').waitFor()`.
- Do not write the test to expect network requests for every location map on open; the intended behavior is on-demand image loading as selection changes.
- For mobile tests, use Playwright's `test.use({ viewport: { width: 375, height: 812 } })` or a named device.
- For backdrop click, click at coordinates outside the content container but inside the overlay.
- Test both trigger placements: hero trigger (desktop) and floating trigger (mobile).
