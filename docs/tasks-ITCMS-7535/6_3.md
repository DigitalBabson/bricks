# Task 6.3 — Add First Keyboard Tab Stop to `BrickCard.vue` (Card Wrapper)

**Phase:** 6 — BrickCard Enhancements
**Spec:** [§3.9.2 Keyboard / Tab Order](../03-features.md#392-keyboard--tab-order)
**Depends on:** Tasks 6.1–6.2

---

## Implementation Details

Make the whole brick card the first keyboard tab stop. This wrapper-level interaction opens the location-details modal for keyboard users and provides the primary focus ring, but mouse clicks should continue to use the visible controls rather than treating the full card as one large click target.

**Current state:** The card does not expose a structured wrapper-level keyboard interaction. The bottom button is clickable, but the card itself is not the first tab stop described in the wireframe.

### Card wrapper behavior

Update the outer interactive container so it becomes the first tab stop:

```html
<article
  class="brick-card tw-shadow-brickCard tw-text-center"
  tabindex="0"
  role="button"
  :aria-label="`View location details for ${brick.inscription}`"
  @keydown.enter.prevent="openMap"
  @keydown.space.prevent="openMap"
>
```

### Focus styling

Add a visible focus indicator that matches the wireframe intent:

- `focus:tw-outline-none`
- `focus:tw-ring-2`
- `focus:tw-ring-brickSummerNight`
- or an equivalent solid high-contrast focus treatment already used by the codebase

### Event behavior

- `Enter` and `Space` on the focused card open the location-details modal.
- Mouse clicks should remain scoped to the visible UI controls:
  - image area → enlarge modal for real-image bricks
  - bottom CTA → location-details modal
- Do **not** turn the whole card wrapper into a mouse click target; that would conflict with the image-click-to-enlarge behavior.
- Inner controls introduced in later tasks still need stopped propagation where appropriate so keyboard/mouse behaviors stay isolated.

### Hover/focus coordination

- The wrapper's keyboard focus should also activate the same white hover wash / enlarge affordance visibility via `group-focus-within`.
- The visible focus ring remains the primary keyboard affordance.

### Coming Soon variant

- The first tab stop still exists for placeholder bricks.
- It opens location details exactly the same way as normal bricks.
- The placeholder image area itself remains non-clickable for mouse users.
