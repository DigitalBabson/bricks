# Task 1.6 — Update `BrickCard.vue` Placeholder Image

**Phase:** 1 — Page Layout
**Spec:** [§3.4 "Coming Soon" Bricks](../03-features.md#34-coming-soon-bricks--show-inscription)

---

## Implementation Details

Update the fallback image URL in `BrickCard.vue` to use the new `DEV_PLACEHOLDER_IMAGE` env var.

**Current state:** `BrickCard.vue` has a hardcoded `defaultImgPath`:

```typescript
data() {
  return {
    defaultImgPath: "/sites/default/files/2025-10/coming-soon.jpg",
  };
}
```

**Changes:**

1. Replace the hardcoded path with `import.meta.env.DEV_PLACEHOLDER_IMAGE`.
2. The new placeholder image is: `https://babsondev.prod.acquia-sites.com/sites/default/files/2026-03/coming-soon-gray.jpg` (dev).
3. This task updates only the image URL. The inscription text overlay ("Image Coming Soon" + brick inscription) is implemented in Phase 6.

**Implementation:**

```typescript
data() {
  return {
    defaultImgPath: import.meta.env.DEV_PLACEHOLDER_IMAGE
      || '/sites/default/files/2026-03/coming-soon-gray.jpg',
  };
}
```

The fallback ensures the component still works if the env var is missing.

**Files to modify:**

- `src/components/BrickCard.vue` — update `defaultImgPath`

## Acceptance Criteria

- [x] `defaultImgPath` reads from `import.meta.env.DEV_PLACEHOLDER_IMAGE`.
- [ ] Bricks without images display the new gray placeholder (`.../coming-soon-gray.jpg`).
- [x] The old `coming-soon.jpg` path is no longer referenced.
- [x] A reasonable fallback exists if the env var is undefined.
- [ ] Bricks with images continue to display their actual photos.
- [ ] No console errors related to missing images.

---

## Findings

- Fallback uses `|| '/sites/default/files/2026-03/coming-soon-gray.jpg'` — the env var takes precedence when available, but the component still works without `.env`.
- The `defaultImgPath` is now a full URL when the env var is set, which would double-prefix if concatenated with `this.env`. Fixed by adding a `fallbackImgUrl` computed property that checks if `defaultImgPath` starts with `http` and skips the prefix if so. All fallback usages in `onImgError`, `getBrickImgURL`, etc. now use `this.fallbackImgUrl`.
