## Implementation Details

Rewrite `src/App.vue` to establish the new page layout: `AppHeader → AppHero (wrapping BrickFilter) → TheBricks → AppFooter`.

**Current state:** `App.vue` wraps a single `<the-bricks/>` component and provides `defaultEnv`/`defaultUrl` via `provide()`.

**New template structure:**

```html
<template>
  <app-header />
  <app-hero @openLocations="showLocationExplorer = true">
    <!-- BrickFilter will be slotted here; for now keep it inside TheBricks
         and move it in Task 1.5 -->
  </app-hero>
  <the-bricks />
  <!-- Mobile-only LocationExplorerTrigger -->
  <location-explorer-trigger
    class="md:tw-hidden"
    :floating="true"
    @openLocations="showLocationExplorer = true"
  />
  <app-footer />
</template>
```

**Key points:**

- Import and register `AppHeader`, `AppHero`, `AppFooter`.
- Add `showLocationExplorer` boolean state (defaulting to `false`). The actual `LocationExplorer` component is built in Phase 5 — for now, just track the state.
- Keep existing `provide()` calls for `defaultEnv` and `defaultUrl` intact (they will be migrated to `import.meta.env.DEV_*` in Phase 3).
- The mobile `LocationExplorerTrigger` is rendered from `App.vue`, hidden on desktop (`md:tw-hidden`), and positioned fixed at the bottom-right of the viewport.

**Files to modify:**

- `src/App.vue` — rewrite template and script

## Findings

- `provide()` was already migrated from Options API to `app.provide()` in `main.ts` during the TypeScript migration. No changes needed.
- Mobile trigger uses `md:tw-hidden` to hide on desktop, matching the hero trigger's `tw-hidden md:tw-block`, and relies on fixed positioning for persistent visibility during scroll.
