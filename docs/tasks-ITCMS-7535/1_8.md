## Implementation Details

Write unit tests for the three new components and update E2E tests to verify the new page structure.

### Unit Tests (Vitest + Vue Test Utils)

Create test files in `src/components/__tests__/`:

**`AppHeader.spec.js`**

- Renders the `<header>` element.
- Contains "BABSON COLLEGE" text.
- Contains "FIND MY BRICK AT KERRY MURPHY HEALEY PARK" text.
- Uses correct Tailwind classes (`tw-bg-brickBabsonGreen`, `tw-text-white`, `tw-font-oswald`).

**`AppHero.spec.js`**

- Renders a section with a background image style.
- Default slot content is rendered inside the hero.
- Desktop `LocationExplorerTrigger` is present.
- Clicking the trigger emits `openLocations`.
- Provide `defaultEnv` and `defaultUrl` via `global.provide` (matching existing test pattern).

**`LocationExplorerTrigger.spec.js`**

- Renders the CTA copy.
- Emits `openLocations` on click.
- Supports the mobile floating variant.

**`AppFooter.spec.js`**

- Renders the `<footer>` element.
- Contains "BABSON COLLEGE" text.
- Does not contain subheader text ("FIND MY BRICK…").
- Uses correct Tailwind classes.

### E2E Tests (Playwright)

Update or add scenarios in `tests/e2e/`:

**Page layout test:**

- Header, hero, brick grid, and footer all render in the correct top-to-bottom order.
- Header contains both lines of text.
- Hero section has a visible background image.
- Footer contains only "BABSON COLLEGE".

**Mobile layout test (viewport < 768px):**

- Mobile location trigger is visible as a floating bottom-right control.
- Mobile location trigger is not visible in the hero area.

**Regression:**

- Existing brick loading, search, and modal functionality still works after the layout restructure.
