## Implementation Details

Write unit tests covering the new location filter behavior in `BrickFilter.vue` and the updated fetch routing in `TheBricks.vue`.

### BrickFilter.vue Tests

Create or update `src/components/__tests__/BrickFilter.spec.ts`:

**Location list rendering:**

- Renders the "Brick Locations" label.
- Renders all location items from the `locations` prop.
- Renders no items when `locations` is an empty array.
- List container has `role="listbox"`.
- Each item has `role="option"` and `aria-selected`.

**Location selection (mouse):**

- Clicking a location emits `update:locationIds` with the full selected-ID array.
- Clicking the already-selected location emits `update:locationIds` with that ID removed from the array.
- Selected location has the active visual class (e.g., `tw-bg-brickLightGreen`).

**Location keyboard navigation:**

- Listbox has `tabindex="0"` and is focusable.
- On focus, `aria-activedescendant` is set to the selected location's option ID (or the first item if nothing is selected).
- Pressing `ArrowDown` moves `activeIndex` to the next item; wraps from last to first.
- Pressing `ArrowUp` moves `activeIndex` to the previous item; wraps from first to last.
- Pressing `Home` moves `activeIndex` to the first item.
- Pressing `End` moves `activeIndex` to the last item.
- Pressing `Enter` on the active option emits `update:locationIds` with that location toggled in the array.
- Pressing `Space` on the active option emits `update:locationIds` with that location toggled in the array.
- Active option has a distinct visual indicator (inset ring) separate from the selected highlight.
- `aria-activedescendant` updates to match the active option after each keyboard move.

**Keyword input (existing + verify no regression):**

- Typing in the keyword input emits `update:inscription`.
- No dedicated inline clear button is rendered inside the keyword field.

**Active filter pills:**

- No pills render when `inscription` and `locationIds` are both empty.
- Keyword pill renders with correct label text when `inscription` is set.
- Location pill renders with the location name (not UUID) when a single `locationIds` value is set, or a multi-select summary when more than one location is selected.
- Both pills render when both filters are active.
- Clicking `×` on keyword pill emits `update:inscription` with empty string.
- Clicking `×` on location pill emits `update:locationIds` with an empty array.
- "Clear all" button emits `clearAll`.
- "Clear all" button is not rendered when no filters are active.

### TheBricks.vue Test Updates

Update `src/components/__tests__/TheBricks.spec.ts`:

**Location-only filtering:**

- When `locationIds` is set, fetch URL includes `filter[brickParkLocation.id][operator]=IN`.
- When `locationIds` is empty, fetch URL does not include location filter.
- `sort=brickInscription` is present in location-filtered requests.

**Page reset on location change:**

- Changing `locationIds` resets `currentPage` to 1.
- Changing `locationIds` triggers a new fetch.

**Empty results:**

- `showMessage` is true when a location filter returns zero results.

## Findings

- Added `App.spec.ts` even though App-level tests were optional, to cover `fetchLocations()` on mount and `clearAllFilters()` state reset.
- The BrickFilter keyboard tests surfaced a real DOM-shim gap (`scrollIntoView` in happy-dom), so the implementation now guards that call safely instead of assuming browser-only behavior in tests.
- The location fixtures were updated to the live JSON:API schema, which exposes the relationship as `brick_zone_image` rather than `field_brick_zone_image`.
- BrickFilter unit coverage now treats keyword reset as a form-level concern and explicitly verifies the inline clear button is absent.
- BrickFilter no longer accepts or renders a result-count summary, so the tests focus only on the visible filter actions.
- Verified after implementation with `npm run lint`, `npm run typecheck`, `npx vitest run src/components/__tests__/App.spec.ts src/components/__tests__/BrickFilter.spec.ts src/components/__tests__/TheBricks.spec.ts`, and `npm test -- --run`.
